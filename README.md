# SPI Protocol Implementation in Verilog

This project implements a **Serial Peripheral Interface (SPI)** communication system using Verilog HDL. The design includes an SPI Master and SPI Slave that communicate using full-duplex serial data transfer.

The SPI master generates the clock and controls the communication, while the slave responds to the master by sending and receiving data simultaneously.

The design is verified through simulation using a testbench.

---

# Project Overview

The implemented system demonstrates:

* SPI Master design
* SPI Slave design
* Full-duplex communication
* Clock division for generating SPI clock
* FSM based control logic
* Shift register based serial data transfer
* Testbench verification with waveform analysis

---

# SPI Communication Signals

The following signals are used for SPI communication:

| Signal   | Description                                          |
| -------- | ---------------------------------------------------- |
| **SCLK** | Serial clock generated by the master                 |
| **MOSI** | Master Out Slave In (data sent from master to slave) |
| **MISO** | Master In Slave Out (data sent from slave to master) |
| **CS**   | Chip Select signal to enable the slave               |

---

# Architecture

The design consists of three main modules.

## 1. SPI Master

Responsible for:

* Generating SPI clock
* Sending data on MOSI
* Receiving data on MISO
* Controlling the chip select signal
* Managing communication using an FSM

Key internal components:

* Clock Divider
* FSM Controller
* Transmit Shift Register
* Receive Shift Register

---

## 2. SPI Slave

Responsible for:

* Receiving MOSI data from the master
* Sending response data through MISO
* Using shift registers to serialize and deserialize data

---

## 3. SPI Top Module

The top module connects the SPI Master and SPI Slave together using the SPI bus signals.

Connections include:

* MOSI
* MISO
* SCLK
* CS

# Data Transfer Example

In simulation:

Master transmits:

```
0x49
```

Slave transmits:

```
0xA5
```

Final result:

```
Slave received  = 0x49
Master received = 0xA5
```

This confirms correct **full duplex SPI communication**.

# Simulation

A testbench is used to verify the design.

The testbench performs the following steps:

1. Generates system clock
2. Applies reset
3. Enables SPI transfer
4. Provides transmit data for master and slave
5. Monitors received data

Simulation output:

```

Slave received  = 49
Master received = A5
```

# Future Improvements

This project can be extended with additional features such as:

### Multi-Slave Support

Support communication with multiple slave devices using separate chip select lines.

Example:

```
CS0 → Slave 0
CS1 → Slave 1
CS2 → Slave 2
```

The master will select the required slave using the corresponding chip select signal.

---

### SPI Mode Support

Currently the design follows **SPI Mode-0**:

```
CPOL = 0
CPHA = 0
```

Future versions can support all four SPI modes:

| Mode   | CPOL | CPHA |
| ------ | ---- | ---- |
| Mode 0 | 0    | 0    |
| Mode 1 | 0    | 1    |
| Mode 2 | 1    | 0    |
| Mode 3 | 1    | 1    |

---

### Configurable Data Width

Support parameterized data width instead of fixed 8-bit communication.

Example:

```
parameter DATA_WIDTH = 8;
```

---

### Configurable Clock Divider

Allow dynamic control of SPI clock frequency.
