# SPI Master–Slave Implementation in Verilog HDL

## 📌 Overview

This project implements a basic **SPI (Serial Peripheral Interface)** Master and Slave communication system in Verilog HDL.

The design demonstrates:
- FSM-based transmission control
- Clock divider-based SCLK generation
- MSB-first serial data transmission
- SPI Mode-1 operation (CPOL = 0, CPHA = 1)

The master initiates communication and sends 8-bit serial data to the slave, which receives and reconstructs the parallel data.

---

## 🧠 SPI Protocol Basics

SPI is a synchronous serial communication protocol used for high-speed communication between microcontrollers and peripheral devices such as ADC, DAC, EEPROM, etc.

### Signals Used:
- **MOSI** – Master Out Slave In
- **MISO** – Master In Slave Out (not implemented in this version)
- **SCLK** – Serial Clock generated by Master
- **CS (Chip Select)** – Active low slave enable

---

## ⚙️ SPI Mode Used

This implementation follows:

- **CPOL = 0** → Clock idle state is LOW
- **CPHA = 1** → Data is sampled on second clock edge

Master updates MOSI on positive edge of SCLK.  
Slave samples MOSI on negative edge of SCLK.

---

## 🏗 Architecture

The design consists of three modules:

### 1️⃣ SPI_Master
- Generates SPI clock using clock divider
- Controls CS signal
- Implements FSM for transmission
- Serializes 8-bit parallel data using shift logic

### 2️⃣ SPI_Slave
- Detects CS low to start reception
- Samples MOSI on appropriate clock edge
- Reconstructs serial data into 8-bit parallel format
- Asserts `done` signal after 8-bit reception

### 3️⃣ SPI_Top
- Integrates Master and Slave for functional simulation

---

## 🔄 Working Flow

1. `tx_enable` asserted by user.
2. Master moves from IDLE → TRANSMIT state.
3. CS is driven LOW.
4. Master shifts out 8-bit data (MSB first).
5. Slave samples incoming bits and shifts into register.
6. After 8 clock cycles:
   - Slave asserts `done`
   - Master deasserts CS
   - FSM returns to IDLE
